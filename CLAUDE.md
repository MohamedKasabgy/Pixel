## Documentation Habit

Keep project docs up to date when a change affects setup, scope, workflows, page maps, QA, release steps, or team handoff.

If updating a document would benefit the team, do it as part of the same task instead of leaving the knowledge only in chat.

Relevant docs include:

- `PROJECT_KNOWLEDGE.md`
- `PIXEL_TASKS.md`
- `README.md`
- `docs/BUILD_PLAN.md`
- `docs/agents/*.md`
- future setup, QA, release, and handoff docs

Do not create noisy docs churn. Update docs when they clarify real project decisions, setup steps, responsibilities, or verification outcomes.
